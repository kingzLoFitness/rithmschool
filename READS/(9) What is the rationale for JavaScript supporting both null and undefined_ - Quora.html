E-Dan Bloch, Developer
Written Apr 25, 2011
One way to consider the notion of ‘undefined’ in JavaScript is as a “higher level” null value. While ‘null’ refers to the content of a variable or the object it points to, ‘undefined’ refers to the state of the variable itself.
The rationale of having both ‘null’ and ‘undefined’ is to allow more flexible and fluent code in JavaScript.  For instance, let’s consider the following example:
var myVar= {“x”:null,”y”:”Why”};

Now:
  • myVar.y is defined and not null
  • myVar.x is defined to be null (hence, not ‘undefined’)
  • myVar.z is undefined and not null.

This gives the programmer more power in understanding the state of an object. More specifically it comes in handy when checking members of an existing object (like x,y,z above).

Another important thing is that while most programming languages throw type errors \exception when an undefined member is accessed (like myVar.z), JavaScript handles this more gracefully and provides the following syntactic sugar:
If (myVar.z===undefined) {…} //runs the {..} segment if myVar has a member named z

And:
var res = myVar.z || myVar.x || myVar.y; // after this line is evaluated, res==”Why”

*Although ‘undefined’ may be used and set as a regular value for variable, just like null, this is considered to be bad practice.

Object is the core data type to JavaScript. Objects are sets of key-value pairs. For each pair, if the key exists, but value does not, it is null. If neither key, nor value exist, it is in the undefined state.
